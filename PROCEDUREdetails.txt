PART B. ##### MAKING the FabMo-RPi-SD-Image (based on downloaded and initialized Working-OS-Image from PART A)
7. Copy the "Working-OS-Image" to an SD-card, with Balena Etcher e.g.(wrk-OS-base-4-19-26-m.img). THIS will now become the SD card FOR our FabMo-RPi-SD-Image BUILD !
8. Boot up this card up on a Raspberry Pi (I have been using RPi5):
    - Go to Preferences > RPi COnfiguration > Interfaces > and turn SSH on if it is not (already is)

9. Open up an ssh terminal connection (pi;shopbot), THEN DO:
    Put a temporary copy of this repo on the pi:  
                       cd /
                       sudo git clone https://github.com/fabmo/FabMo_RPi_SD_Image_Builder Image_Builder
    Then:
    - Switch to WinSCP to make a little easier (needs to be one you have set up as user pi for copying, etc)
    - /home/pi/Scripts should exist, but make it fully writable/executable with WinSCP properties
    - copy and paste into Scripts from "Image_Builder" current local repo:
	    resources [folder]
      	build-fabmo-image.sh
    Then make this last file executable by anyone, 777    
10. NEXT DO: 
	From Terminal:
                        cd /home/pi/Scripts  
	                    sudo ./build-fabmo-image.sh

12. WHEN the script is finished running, check to make sure things got installed and permissions look right.
    -Run boot diagnostic:  /home/pi/Scripts/resources/check-boot-config.sh
       (This checks that all RPi 5 boot display settings are correctly applied)
    -Run AP services diagnostic:  /home/pi/Scripts/resources/check-ap-services.sh
       (This checks that setup_wlan0_ap.service is enabled - critical for AP SSID with IP)
    -If boot display or login issues occur, run: sudo /home/pi/Scripts/resources/fix-cold-boot-and-login.sh
    -If AP SSID doesn't show IP, run: sudo /home/pi/Scripts/resources/fix-ap-ssid.sh
       (This enables setup_wlan0_ap.service - run BEFORE cleanup!)

13. Run cleanup script and do additional UI adjustments:
    -Run cleanup:  sudo /home/pi/Scripts/resources/cleanup-build.sh
       (This removes BUILD SCRIPT, RESOURCES folder, and TEMP Repo automatically)
       **NOTE: Run diagnostics BEFORE cleanup as it removes the resources folder**
    -UI Adjustments:
       * Set Task Bar: position bottom, size medium
       * Set Desktop text color to dark blue #353A92
       * Add RPI-CONNECT to Menu Bar (add on right, then move to left side)
    -Verify version file:  cat /boot/fabmo-release.txt
    
    NOTE: The boot sequence has been optimized for RPi 5 (both cold and warm boots):
          - Rainbow splash disabled (disable_fw_kms_setup=1 in config.txt)
          - Firmware splash disabled (disable_splash=1)
          - Boot delay minimized (boot_delay=0 for faster cold boot)
          - FabMo logo displays via Plymouth for ~3-5 seconds
          - Console suppressed (loglevel=3, console=tty3)
          - Boot parameters maintained by fabmo-boot-params.service
          - Auto-login configured for user pi (no login prompt)
          - Works consistently on both power-on (cold) and reboot (warm)
          - You should NEVER see a login prompt
    
    IMPORTANT - RPi 5 EEPROM Configuration (Per-Board, Not Per-Image):
          Newer RPi 5 boards (especially 8GB models) may show a white/red "Install an OS" 
          screen on first cold boot. This is a board EEPROM setting, not an SD card issue.
          
          TO FIX (one-time per board):
          1. After first boot completes, run: sudo raspi-config
          2. Navigate to: Advanced Options → Network Install UI → Disable
          3. Reboot - white/red screen should be gone permanently
          
          This setting is stored in the board's hardware EEPROM, so it only needs to be 
          done once per physical board, not every time you flash a new image.
    


14. DO NOT Reboot this card (e.g. as a test) if you want to make a small version as it seems to expand for the last time on the reboot (see cmd.txt init=)!
    IMPORTANT: The firstboot script will rewrite cmdline.txt on first expansion, but the fabmo-boot-params.service 
               will automatically restore boot display settings on every boot to prevent issues. 
15. Make a copy of this SD card to a smaller SD card; 16G seems to work at the moment using SD Card Copier Accessory in RPi.
16. This SD card can now be used with the Win32Disk Imager (GOOGLE DOCS OFF) to make our NEW FabMo-RPi-Image

17. Make a Zipped Copy of the NEW FabMo-RPi-Image for distribution and post.



=======================================
POST-INSTALLATION NOTES FOR END USERS
=======================================

When distributing the FabMo image, include these instructions for first-time setup:

** FOR RPi 5 BOARDS (especially 8GB models) **

If you see a white/red screen with "Install an OS" text on first boot:
  1. Wait for boot to complete (may take 2-3 minutes)
  2. Once desktop appears, open Terminal
  3. Run: sudo raspi-config
  4. Navigate to: Advanced Options → Network Install UI → Disable
  5. Exit and reboot
  6. White/red screen should be gone permanently

This is a one-time hardware configuration (stored in board EEPROM, not on SD card).
Older RPi 5 boards may not have this issue.

** EXPECTED BOOT SEQUENCE (after EEPROM fix if needed) **
  - Black screen (1-2 seconds)
  - FabMo logo (3-5 seconds)
  - ShopBot desktop (auto-login, no password prompt)


** DISTRIBUTION PACKAGE **

Include FIRST_BOOT_SETUP.txt with the zipped image file for end-user reference.
This file provides simple instructions for the EEPROM configuration if needed.



Some details:
    = The BUILD script creates symlink for many of the systemd service files that FabMo uses and some scripts. This is done so that the
         functionality that these files support can be managed in the fabmo repo, along with fabmo. Thus if they are updated a full new image build is
         not required as they will be updated along with fabmo (or fabmo-updater). This means one needs to attend to keeping
         these files current in fabmo and fabmo-updater. Currently they are:
               /fabmo/files:
                   fabmo.service
                   camera-server-1.service
                   camera-server-2.service
                   usb_logger.service
                 /fabmo/files/network_conf_fabmo:
                     network-monitor.sh
                     network-monitor.service
                     setup_wlan0_ap.service  **CRITICAL for AP SSID with IP address**
                     export-netcfg-thumbdrive.service
                     export-netcfg-thumbdrive.path
                     fabmo-ip-reporting.desktop (autostart for desktop IP display)
               /fabmo-updater/files:
                   fabmo-updater.service
               /usr/local/bin:
                   ck_heat_volts.sh
                   ck_network.sh
                   ck_services.sh
                   ld_firmware.sh
               /sysd-services (from this repo's resources):
                   hostapd.service
               /NetworkManager (from this repo's resources):
                   /system-connections:
                           direct-connection 
                           lan-connection
                           wlan0_ap.nmconnection 
                   NetworkManager.conf    


    *Note: modify-priorities and set-network-priorities are no longer being used
    *It is possible that a systemctl reload-daemon may be needed for symlinks whose content is altered!
